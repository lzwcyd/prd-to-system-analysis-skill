# CURRENT_STATE（工程现状盘点）

本文件仅定义 CURRENT_STATE 阶段规则。固定输出骨架请使用 `templates/current-state.md`，通用提示块请使用 `templates/zh-CN.md` 或 `templates/en.md`。

## 目标

基于工程证据确认“现在是什么样”，为方案设计提供可验证边界。

## 阶段规则

- 必须读取工程现状，不得凭空假设已有能力。
- 优先检查：
  - `README`、`docs`、接口文档
  - 关键服务、领域模型、DAO/仓储
  - 现有接口路由、请求/响应模型、鉴权/验签规范
  - 数据层（表结构、迁移脚本、实体关系）
  - 非功能（监控、日志、限流、任务、回归）
- 输出 `现状能力清单`，`与 PRD 关系` 仅可使用：
  - `复用` / `改造` / `新增` / `缺失`
- 证据必须可追溯到具体文件或模块路径。
- 若 PRD 涉及字段新增、类型扩展或口径变化，必须补充“历史逻辑影响”盘点：
  - 历史数据分布与默认值风险
  - 旧任务/旧查询/旧接口受影响面
  - 需要改造或隔离的历史链路

## 规则密集需求增强项

若需求属于公式/规则密集型（如费率、减免、容差），必须补充独立规则矩阵：

- 公式/逻辑
- 输入字段
- 边界条件
- 特殊场景
- 告警/兜底策略

## 输出与落盘

1. 按当前 `lang` 使用 `templates/current-state.md` 对应语言骨架组织输出。
2. 写入 `artifact_root/current-state.<lang>.md`。
3. 更新 `artifact_root/analysis.state.<lang>.json`（至少包含 `artifacts.current_state`）。

## 最小校验清单

- 是否给出关键能力证据
- 是否完成“与 PRD 关系”映射
- 是否识别并记录当前缺口
- 若有数据演进，是否识别历史兼容风险
- 是否完成阶段确认并得到用户同意

## 与下游衔接

CURRENT_STATE 确认后进入 `PLAN`，输出至少 A/B 两个方案并要求用户选择。
